# checkFood

平板点餐记录工具，面向后厨实时使用。它用于快速记录多组客人的点菜内容、备注和费用，不做历史订单、不做权限管理，重点是点得快、看得清、金额能自动计算。

## 功能

- 点餐页左侧按分类展示菜品：热菜、凉菜、饮料。
- 点击菜品后，在左侧底部固定备注区选择备注并确认添加；连续点击同一道菜会生成多条记录，方便后厨逐份查看。
- 右侧按第 1 组、第 2 组展示菜单，每组单独计算小计，顶部展示所有组的合计金额。
- 菜品提交后仍可点击对应组的“加菜”，继续把新菜加到该组。
- 点击右侧某一道已点菜品，可以继续修改这道菜的备注。
- 后厨完成某道菜后点击“完成”，该菜会变成绿色；也可以取消完成状态。
- 删除菜单前会弹出确认，删除后不保留历史记录。
- 管理页支持新增、编辑、删除菜品，并支持新增、删除固定备注。
- 数据保存在浏览器 `localStorage` 中，刷新页面后仍保留当前实时菜单。

## 使用流程

1. 打开点餐页。
2. 点击左侧菜品。
3. 在左侧底部固定区域选择备注，点击“确认添加”。
4. 一组客人点完后，点击右侧该组的“提交”。
5. 新来一组客人时，点击左侧“新建一组”。
6. 如果某组后续加菜，点击该组的“加菜”，再从左侧选择菜品。
7. 厨师完成某道菜后，在右侧该菜上点击“完成”。

## 本地运行

推荐使用 Node.js 20 或更高版本。

```bash
npm install --include=optional --cache .npm-cache --registry https://registry.npmmirror.com
npm run dev -- --host 127.0.0.1
```

启动后按终端提示访问本地地址，通常是：

```text
http://127.0.0.1:5173/
```

## 测试和构建

```bash
npm test -- --run
npm run build
```

## 常见问题

如果启动时报错 `Cannot find module @rollup/rollup-darwin-arm64`，通常是 npm 可选依赖没有安装完整。可以重新执行：

```bash
npm install --include=optional --cache .npm-cache --registry https://registry.npmmirror.com
```

如果安装时报 `EACCES` 并提示 `~/.npm` 权限问题，优先使用上面的 `--cache .npm-cache`，让 npm 缓存写入项目目录，避免使用当前用户没有权限的全局缓存目录。
